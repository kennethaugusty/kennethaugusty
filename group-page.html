    The Project

    The core of the project is the Data Migration from one database to another databse.

    The Source is provided, we the group members have to analyse this data and find a way to migrate the data to a target database we choose after extracting and transforming the data.

    In this Scenario, we are provided with The Northwind Database as the Source.

    The Northwind database is a database which is built by Microsoft.

    The task we have been given is to find a way to create a data warehouse with the data provided.

    The Source ER diagram is shown here.




    In order to create a data warehouse from all the different methods we chose to follow a STAR approach to create the data warehouse.

    On further observation of the data, the important data from the data is retained. The 13 tables from the source was brought in together to create the warehouse which contains essential data and the necessary data can be fetched or looked up without much hassle.

    The 13 tables was clubbed together as 5 tables to make our target.

    1) Product - contains all the details of the product.
    2) Supplier - The one that provides the product.
    3) Customer - The one buying the product.
    4) Employee - details of the one from whom the product is being transacted.
    5) Fact table - The culmination of all the other 3 tables, contains details of the orders recieved and served with various details regarding the order.

    The Target ER diagram is shown here.




    The Source database that we have used is MySQL

    The Target database that we have used is PostgreSQL

    The connection between MYSql and PostgreSQL is done through an Python IDE. The IDE used here is Jupyter Notebook. The data is taken from sql and stored in a pandas dataframe, the data in this data frame is then taken for transforming and processing, after all the steps the data is then sent to the target.

    The connection is established using SQLAlchemy between the databases.

    We take in the required columns, make our prefered changes and then write them to the pre made tables in postgres.


    The steps:

    1)    Open connection to source​

    2)    Extract data from the Source​

    3)    Store the data in a dataframe

    4)    Transform the values as required​

    5)    Type conversion as needed by PostgreSQL

    6)    Match the column of the dataframe and the
           pre existing postgre table​

    7)    Open connection to target

    8)    Load data from data_frame to the target table

    9)    Close connection of source and target

    10)  End​
